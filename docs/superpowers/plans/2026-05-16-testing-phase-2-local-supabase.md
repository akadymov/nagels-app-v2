# Testing strategy — Phase 2 (Local Supabase + isolated Expo) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a fully isolated test backend — local Supabase (Docker, port `54321`) plus a dedicated Expo dev server on port `:8082` reading a generated `.env.test` — orchestrated automatically by Playwright's `globalSetup` / `globalTeardown`. The existing SP e2e spec must pass against this stack, while Akula's manual `:8081` dev server stays untouched.

**Architecture:** Phase 2 adds a Playwright-side "local backend" mode toggled by `LOCAL_SUPABASE=1`. When the flag is set, `globalSetup` (a) runs `supabase start` (idempotent — no-op if already up), (b) applies all migrations via `supabase db reset --local --no-seed`, (c) parses `supabase status -o json` for URL/anon-key, (d) refuses to proceed if the URL is not `127.0.0.1`/`localhost` (prod-leak guard), (e) writes a gitignored `.env.test`, (f) spawns `npx expo start --port 8082` with the local URL/key injected into the env, and (g) waits for the bundler to respond on `:8082`. `globalTeardown` kills the Expo child and runs `supabase stop --no-backup` unless `KEEP_SUPABASE=1`. When `LOCAL_SUPABASE` is not set, both hooks no-op — the existing `npm run test:sp` (against the manual `:8081` dev server) keeps working byte-for-byte.

The Supabase-status parsing and URL-guard logic are tiny pure helpers extracted into `tests/playwright/local-backend.ts` and exercised by Jest unit tests. Everything else lives in the two glue files `global-setup.ts` and `global-teardown.ts`.

**Tech Stack:** Supabase CLI 2.95.4 (local dev: Postgres + Realtime + edge-runtime in Docker), Expo (`npx expo start --port 8082`), Playwright `globalSetup`/`globalTeardown`, Node `child_process.spawn`, Jest + ts-jest for the helper unit tests, TypeScript end-to-end.

Reference spec: `docs/superpowers/specs/2026-05-16-testing-strategy-design.md` — sections "Test backend — local Supabase via CLI", "Rollout phases — Phase 2 (Local Supabase + isolated Expo)", and the "Test-data leakage to prod" mitigation.

Phase 1 plan (already shipped): `docs/superpowers/plans/2026-05-16-testing-phase-1-foundation.md`.

```ts
import * as path from 'path';

export interface SupabaseStatus {
  apiUrl: string;
  anonKey: string;
  serviceRoleKey: string;
  dbUrl: string;
}

/**
 * Parse the JSON emitted by `supabase status -o json`. Throws a clear
 * error when a required field is missing so a future CLI version that
 * renames a key fails loudly instead of silently writing an empty
 * .env.test.
 */
export function parseSupabaseStatus(raw: string): SupabaseStatus {
  const j = JSON.parse(raw) as Record<string, unknown>;
  const need = (k: string): string => {
    const v = j[k];
    if (typeof v !== 'string' || v.length === 0) {
      throw new Error(
        `supabase status JSON missing required field "${k}". ` +
          `Got: ${JSON.stringify(j)}`,
      );
    }
    return v;
  };
  return {
    apiUrl: need('API_URL'),
    anonKey: need('ANON_KEY'),
    serviceRoleKey: need('SERVICE_ROLE_KEY'),
    dbUrl: need('DB_URL'),
  };
}

/**
 * Guard against accidentally pointing tests at a remote supabase
 * project. Any deviation from 127.0.0.1 / localhost aborts the run
 * before any seed inserts or migration resets touch real data.
 */
export function assertLocalUrl(url: string): void {
  const isLocal =
    url.startsWith('http://127.0.0.1') ||
    url.startsWith('http://localhost') ||
    url.startsWith('https://127.0.0.1') ||
    url.startsWith('https://localhost');
  if (!isLocal) {
    throw new Error(
      `refusing to run: supabase URL "${url}" is not local. ` +
        `Tests must only touch 127.0.0.1 / localhost backends.`,
    );
  }
}

export interface RuntimePaths {
  envTest: string;
  runtimeDir: string;
  expoPid: string;
  expoLog: string;
}

/**
 * Centralizes path resolution so setup, teardown, and tests all
 * agree on where the runtime scratch files live.
 */
export function runtimePaths(projectRoot: string): RuntimePaths {
  const runtimeDir = path.join(projectRoot, 'tests', '.runtime');
  return {
    envTest: path.join(projectRoot, '.env.test'),
    runtimeDir,
    expoPid: path.join(runtimeDir, 'expo.pid'),
    expoLog: path.join(runtimeDir, 'expo.log'),
  };
}
```

```ts
import { spawn, execSync } from 'child_process';
import * as fs from 'fs';
import * as path from 'path';
import * as http from 'http';
import {
  parseSupabaseStatus,
  assertLocalUrl,
  runtimePaths,
} from './local-backend';

const PROJECT_ROOT = path.resolve(__dirname, '..', '..');
const EXPO_PORT = 8082;
const EXPO_READY_TIMEOUT_MS = 120_000; // bundler cold start can be slow
const EXPO_POLL_INTERVAL_MS = 1_500;

/**
 * Playwright `globalSetup`. Activated by env LOCAL_SUPABASE=1; no-op
 * otherwise so the legacy `npm run test:sp` flow against the manual
 * :8081 dev server keeps working unchanged.
 */
export default async function globalSetup(): Promise<void> {
  if (process.env.LOCAL_SUPABASE !== '1') {
    return;
  }

  const paths = runtimePaths(PROJECT_ROOT);
  fs.mkdirSync(paths.runtimeDir, { recursive: true });

  // 1. Idempotent supabase start. `supabase status` exits non-zero
  //    when the stack isn't running, so we use it as the probe.
  if (!isSupabaseUp()) {
    log('starting local supabase…');
    execSync('supabase start', { stdio: 'inherit', cwd: PROJECT_ROOT });
  } else {
    log('supabase already up (KEEP_SUPABASE iteration)');
  }

  // 2. Apply all migrations on a clean DB. --no-seed because the
  //    project doesn't ship a seed.sql and we don't want to fail
  //    on a missing file.
  log('applying migrations via supabase db reset…');
  execSync('supabase db reset --local --no-seed --yes', {
    stdio: 'inherit',
    cwd: PROJECT_ROOT,
  });

  // 3. Read URL + keys and guard against pointing at prod.
  const statusJson = execSync('supabase status -o json', {
    cwd: PROJECT_ROOT,
  }).toString();
  const status = parseSupabaseStatus(statusJson);
  assertLocalUrl(status.apiUrl);

  // 4. Write .env.test. Expo's spawn env will override .env.local,
  //    but writing the file too means a developer can `cat .env.test`
  //    when debugging a failed run.
  const envTest = [
    `EXPO_PUBLIC_SUPABASE_URL=${status.apiUrl}`,
    `EXPO_PUBLIC_SUPABASE_ANON_KEY=${status.anonKey}`,
    `EXPO_PUBLIC_APP_URL=http://localhost:${EXPO_PORT}`,
    '',
  ].join('\n');
  fs.writeFileSync(paths.envTest, envTest, 'utf8');

  // 5. Spawn the isolated Expo dev server. Inline env vars override
  //    .env.local — Expo's loader precedence is process.env → .env.local
  //    → .env. We also pass CI=1 so the bundler skips interactive
  //    prompts (Open in browser?, Press w to open web, etc.).
  log(`spawning expo on :${EXPO_PORT}…`);
  const logFd = fs.openSync(paths.expoLog, 'w');
  const child = spawn('npx', ['expo', 'start', '--port', String(EXPO_PORT)], {
    cwd: PROJECT_ROOT,
    detached: true,
    stdio: ['ignore', logFd, logFd],
    env: {
      ...process.env,
      EXPO_PUBLIC_SUPABASE_URL: status.apiUrl,
      EXPO_PUBLIC_SUPABASE_ANON_KEY: status.anonKey,
      EXPO_PUBLIC_APP_URL: `http://localhost:${EXPO_PORT}`,
      CI: '1',
    },
  });
  if (typeof child.pid !== 'number') {
    throw new Error('expo failed to spawn (no pid)');
  }
  fs.writeFileSync(paths.expoPid, String(child.pid), 'utf8');
  child.unref();

  // 6. Poll until Expo responds on :8082. The bundler serves a 200
  //    on `/` once the JS bundle is ready, but during cold start it
  //    may serve 503/HTML-with-error-overlay. We accept any 2xx/3xx
  //    as "ready enough" — Playwright tests will fail loudly if the
  //    actual app didn't boot.
  log('waiting for expo to be ready…');
  await waitForHttp(`http://localhost:${EXPO_PORT}`, EXPO_READY_TIMEOUT_MS);
  log('expo ready.');
}

function isSupabaseUp(): boolean {
  try {
    execSync('supabase status', {
      cwd: PROJECT_ROOT,
      stdio: 'ignore',
    });
    return true;
  } catch {
    return false;
  }
}

function waitForHttp(url: string, timeoutMs: number): Promise<void> {
  const deadline = Date.now() + timeoutMs;
  return new Promise((resolve, reject) => {
    const tick = (): void => {
      const req = http.get(url, (res) => {
        res.resume();
        const status = res.statusCode ?? 0;
        if (status >= 200 && status < 400) {
          resolve();
        } else if (Date.now() > deadline) {
          reject(new Error(`expo readiness timeout at ${url} (last status ${status})`));
        } else {
          setTimeout(tick, EXPO_POLL_INTERVAL_MS);
        }
      });
      req.on('error', () => {
        if (Date.now() > deadline) {
          reject(new Error(`expo readiness timeout at ${url} (no response)`));
        } else {
          setTimeout(tick, EXPO_POLL_INTERVAL_MS);
        }
      });
      req.setTimeout(EXPO_POLL_INTERVAL_MS, () => req.destroy());
    };
    tick();
  });
}

function log(msg: string): void {
  // Visible in Playwright's reporter output even with reporter:'list'.
  // eslint-disable-next-line no-console
  console.log(`[global-setup] ${msg}`);
}
```

Create `tests/playwright/global-teardown.ts` with this exact content:

```ts
import { execSync } from 'child_process';
import * as fs from 'fs';
import * as path from 'path';
import { runtimePaths } from './local-backend';

const PROJECT_ROOT = path.resolve(__dirname, '..', '..');

/**
 * Playwright `globalTeardown`. No-op unless LOCAL_SUPABASE=1.
 * KEEP_SUPABASE=1 keeps the docker stack running so subsequent
 * iteration runs skip the ~30s start cost.
 */
export default async function globalTeardown(): Promise<void> {
  if (process.env.LOCAL_SUPABASE !== '1') {
    return;
  }

  const paths = runtimePaths(PROJECT_ROOT);

  // 1. Kill the expo dev server. SIGTERM first; if it's still alive
  //    after 5s, SIGKILL. `unref()` in setup means the child
  //    survives the parent, so the PID file is the only handle.
  if (fs.existsSync(paths.expoPid)) {
    const pid = parseInt(fs.readFileSync(paths.expoPid, 'utf8').trim(), 10);
    if (Number.isFinite(pid) && pid > 0) {
      log(`stopping expo (pid ${pid})…`);
      try {
        process.kill(pid, 'SIGTERM');
      } catch (e: unknown) {
        log(`SIGTERM failed (already gone?): ${(e as Error).message}`);
      }
      await sleep(5000);
      try {
        process.kill(pid, 0); // probe — throws if dead
        log('expo still alive, sending SIGKILL');
        try {
          process.kill(pid, 'SIGKILL');
        } catch {
          /* race: dead between probe and kill */
        }
      } catch {
        /* dead already, good */
      }
    }
    try {
      fs.unlinkSync(paths.expoPid);
    } catch {
      /* fine */
    }
  } else {
    log('no expo pid file — skipping expo kill');
  }

  // 2. Stop supabase unless the iteration flag is set.
  if (process.env.KEEP_SUPABASE === '1') {
    log('KEEP_SUPABASE=1 — leaving local supabase running');
  } else {
    log('stopping local supabase…');
    try {
      execSync('supabase stop --no-backup', {
        cwd: PROJECT_ROOT,
        stdio: 'inherit',
      });
    } catch (e: unknown) {
      log(`supabase stop failed: ${(e as Error).message}`);
    }
  }
```

- [ ] **Step 3: Make Playwright able to load `.ts` global hooks**

Playwright 1.58 supports `.ts` setup files only when the project ships a `playwright`/`@playwright/test` config that registers a TS transform. The project's root `tsconfig.json` extends `expo/tsconfig.base`, which emits `module: esnext` — `require()` cannot load that. The simplest path that avoids touching the global tsconfig is to add a CommonJS shim that registers ts-node with explicit compiler options forcing CJS output.

Replace the `require.resolve('./tests/playwright/global-setup.ts')` line with `require.resolve('./tests/playwright/global-setup')` (no extension) and create `tests/playwright/global-setup.js`:

```js
// Thin CommonJS shim so playwright.config.js can wire a .ts file as
// globalSetup without us needing ts-jest at runtime. We force
// module=commonjs because the root tsconfig extends expo/tsconfig.base
// which emits esnext — require() would reject that.
require('ts-node').register({
  transpileOnly: true,
  compilerOptions: {
    module: 'commonjs',
    target: 'es2020',
    esModuleInterop: true,
    moduleResolution: 'node',
    allowImportingTsExtensions: false,
  },
});
module.exports = require('./global-setup').default;
```

```js
    moduleResolution: 'node',

});
```
